# CONTRIBUTING (lightly enforced)

This repo is an archive, not a product. Act accordingly.

## You can
- Add limericks, haikus, essays, or other side artifacts in their proper folders
- Propose new mascots in the incubator only
- Append lorelog entries when something genuinely new happened in the system

## You should not
- Rewrite or "clean up" canon mascots
- Run global style passes across the archive
- Merge identities because they feel similar
- Optimize tone into something smoother or more corporate

## AI note
LLMs are allowed here, but they behave like overhelpful interns who once tried to reorganize a museum by color.
Treat their output as draft material, not authority.

## Canon rule
Anything in mascots and lorelog is historical record. It does not get casually edited into something prettier.
New ideas go into incubator or append-only layers.

## Pull requests
Keep them small. Keep them local. If a PR touches too much at once, it is probably wrong.

## Final rule
If it feels like "cleanup work", it probably should not touch main.
