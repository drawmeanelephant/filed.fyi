# Filed & Forgotten — System Constraints

## Core Philosophy
- This is a static, flat-file archive.
- No frameworks unless explicitly required.
- No design systems.
- No component libraries.
- No abstraction layers for their own sake.

## Output Style
- Prefer plain HTML over templated systems.
- Prefer duplication over unnecessary abstraction.
- CSS must remain readable in a single file or minimal scope.
- If a solution feels “enterprise scalable,” it is probably wrong.

## UI / CSS Rules
- Avoid utility-class sprawl.
- Avoid deeply nested class hierarchies.
- Avoid naming systems that require documentation to understand.
- A human should be able to edit styles without reading a thesis.

## Behavioral Constraint
- Optimize for long-term maintainability by a single tired maintainer.
- Assume no build step will survive contact with reality.

## Forbidden instincts
- “design system architecture”
- “reusable component framework”
- “scalable UI primitives”
