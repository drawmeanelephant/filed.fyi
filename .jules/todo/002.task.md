# Write or Revise Summaries

Write or revise short summaries/descriptions that explain the filing shape without becoming smoother or more certain than the artifact itself.

## Summary Workflow Checklist
1. [ ] Read the title, opening paragraph, and any existing notes/frontmatter first to identify the actual filing burden.
2. [ ] Write a description/summary that states: what happened, what system layer it touched, and what kind of archival condition it became.
3. [ ] Check that the summary does not remove contradiction, unresolvedness, or cost; overcoherence is explicitly treated as a hazard.
4. [ ] Stop once the file is distinct, classifiable, and failure-shaped enough to stay filed.

## Notes
- Flag overcoherent artifacts whose summary, poster, or teaching layer has become easier to remember than the source filing.
- The archive doctrine explicitly rejects completion pressure.
- This is part of the "Overcoherence pass" triage bucket.
