# Normalize Identity Fields

Normalize title, slug, description, and date on every touched file so identity fields stop drifting and remain cross-referenceable.

## Metadata Checklist
- [ ] `title`: Make it specific, archivally flavored, and stable across references.
- [ ] `slug`: Keep it canonical and aligned with the title/case identity; avoid alias drift.
- [ ] `description`: Write a 1-sentence filing summary that names the condition without over-resolving it.
- [ ] `date` / `updatedAt`: Preserve chronology so incidents and doctrine accrete properly.
- [ ] `status`: Keep filing posture consistent, especially public/internal/restricted and filed/pending/resolved patterns.

## Notes
- This is part of the "Identity pass" triage bucket.
