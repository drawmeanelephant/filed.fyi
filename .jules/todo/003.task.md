# Taxonomy and Tag Cleanup

Clean tags into stable clusters; prefer a few durable tags over decorative sprawl, but preserve canon residue when it still carries meaning.

## Taxonomy Checklist
- [ ] `tags`: Consolidate into useful retrieval clusters; avoid empty, fragmented, or ornamental tags. Preserve legacy drift when it is already canon and not actively obstructive.
- [ ] `concepts`: Use for doctrine-level grouping when the file participates in a larger idea cluster.
- [ ] `classification` / `status`: Verify these fields where they are part of the local schema (especially lorelog and reference objects). Keep filing posture consistent.
- [ ] `systemAffiliation`: Name the desk, cluster, or archive layer the item belongs to so it can be shelved.
- [ ] `managed-absence`: Mark managed-absence cases consistently when the object is “gone but still governing,” using the project’s absence doctrine rather than ad hoc wording.
