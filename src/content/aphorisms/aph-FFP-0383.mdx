<Broside type="tip" title="Decay Metric" icon="trending-down" id="ffp383-aph-1">
Systemic decay is not a failure of the architecture; it is a highly predictable schedule of obsolescence.
</Broside>

<Broside type="tip" title="Exhibit Axiom" icon="trending-down" id="ffp383-aph-2">
The metric of decay only matters if the decline can be represented by a perfectly smooth, uninterrupted line.
</Broside>

<Broside type="tip" title="Rot Calibration" icon="trending-down" id="ffp383-aph-3">
If the system rots faster than the dashboard can report it, you must file a request to slow down the rust.
</Broside>

<Broside type="tip" title="Decay Reality" icon="trending-down" id="ffp383-aph-4">
We do not fix the decay. We simply update the documentation to redefine the decay as the new baseline standard.
</Broside>

<Broside type="tip" title="Measurement Fact" icon="trending-down" id="ffp383-aph-5">
A system is fully decayed when there is no one left to read the error logs it continues to produce.
</Broside>
