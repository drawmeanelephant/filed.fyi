## Explanation Layer Residue Audit

### Purpose

This audit identifies when an explanation stack has become collectively smooth
in a way that individual layer review cannot detect.

It is not a restatement of FREF-0820-IELS (Institutional Explanation Layer
Stabilization), which concerns a single explanation layer that has become
more internally consistent than the event it claims to carry.

It is not a restatement of FREF-0130-OCVH or FREF-0290-OCVH (OverCoherence
Vault and Handling), which concerns a single narrative surface that has
resolved contradictions it should have preserved.

This audit addresses a stacked case: multiple explanation layers, each
individually defensible, each passing its contradiction to the next, until
the event becomes fully legible at every level and the original contradiction
has been distributed so evenly that it cannot be located in any single layer.

---

### Three Conditions, Three Shelves

**Condition A — Single-Layer Over-Coherence (FREF-0820-IELS)**
One explanation is smoother than the event it claims to carry.
The contradiction has been compressed into a single, too-coherent surface.
Filing target: FREF-0820-IELS.

**Condition B — Over-Coherence Vault (FREF-0130-OCVH / FREF-0290-OCVH)**
Multiple systems agree too cleanly. The agreement itself is the failure signal.
A contradiction that should be producing visible friction is instead rendering
as a stable consensus.
Filing target: OverCoherence Vault.

**Condition C — Explanation Layer Residue (this audit)**
Multiple explanation layers exist for the same event.
Each layer is internally consistent.
No single layer contains the contradiction.
The contradiction has been distributed as residue across the stack.
**The event is fully legible at every layer. The problem is not visible in any one of them.**

---

### Trigger Conditions

Run this audit when:

1. An event has generated three or more explanation layers — executive summary,
   audit brief, lorelog record, assurance vocabulary rendition, or equivalent.
2. Each layer is internally coherent and individually auditable.
3. No single layer appears to have smoothed anything impermissibly.
4. The event is nonetheless more legible in its explained form than it was
   at the time of occurrence.
5. The original contradiction cannot be cleanly extracted from any single layer.

If all five conditions apply, the event may be subject to **distributed
contradiction**: each layer offloaded the friction to the next, and the
stack collectively absorbed it.

---

### Distinguishing Test

The multi-layer residue case is present when:

- Removing any single layer from the stack does not restore the contradiction.
- The event's difficulty is recoverable only by reading all layers simultaneously,
  which no routine review process does.
- Layers cite one another for coherence rather than for evidence.
- The bottom layer (lorelog, annex, or restricted record) is the only one with
  the original language — and its citation rate is low.

If the distinguishing test passes, this is a custody and routing audit,
not a quality-of-explanation audit.

### Routing Boundary Rules

| Symptom | Filing Target |
|---|---|
| Single explanation too smooth | FREF-0820-IELS |
| Multiple systems agree where they should conflict | OverCoherence Vault (FREF-0290-OCVH) |
| Contradiction distributed across explanation stack | FREF-0822-ELRA (this audit) |
| Contradiction preserved in annex while headline stays calm | LLG-0861-ARC |
| Explanation improved tone without changing routing | FREF-0630-CWLV |

---

### Audit Steps

1. Collect all explanation layers generated for the event in chronological order.
2. Identify where each layer received the contradiction from the layer below it.
3. Identify where each layer passed it to the layer above.
4. Record whether the contradiction is intact at any single layer, or whether
   it has been distributed.
5. Flag the transfer point — the layer at which contradiction became structural
   residue rather than a named object.

The transfer point is the audit's primary finding. It is not a person, office,
or bad actor. It is the layer at which the contradiction became harder to
locate than to bypass.

---

### Handling Guidance

When a distributed contradiction is identified:

1. Preserve all layers intact.
2. File a comparative note naming which layers participate.
3. Attach a reconciliation note identifying the transfer point.
4. Prohibit the executive layer from citing the lorelog layer as transparency
   evidence unless the lorelog material has been surfaced, not merely retained.
5. Mark the event: *Contradiction distributed across explanation stack — not
   locatable at single layer.*

*Compression warning:* Summary surfaces may inherit smoothness from stack
accumulation rather than from any single layer's deliberate rendering choice.

---

### What This Audit Does Not Govern

- It does not govern the quality or sincerity of any individual explanation layer.
- It does not claim that multi-layer reporting is inherently distorting.
- It does not assign intent to the smoothing.

The smoothing is structural. Each layer did what its local standard required.
The failure is in the aggregate, not the individual output.

---

### Approved Language

**Preferred phrases:**
- multi-layer explanation residue
- contradiction distributed across stack
- no single layer holds the dissonance
- stack-accumulated coherence, not single-point compression
- comparative reading required to locate original contradiction
- each layer outsourced its contradiction to the next

**Disallowed phrases:**
- each layer is accurate therefore no problem exists
- the contradiction was addressed somewhere in the stack
- layered explanation confirms transparency
- the record is complete because every layer is present

- **FREF-0820-IELS** — Institutional Explanation Layer Stabilization: single-layer stabilization; this audit is the multi-layer extension
- **FREF-0290-OCVH** — OverCoherence Vault: single-surface smoothing; distinct from stacked distribution
- **LLG-0861-ARC** — Annex Retention Conscience Event: related pattern in which contradiction survives in annexes while main surfaces remain undisturbed
- **LLG-0812-CTM** — Continuity Theatre Maintenance Window: single-event case study in explanation stack management
- **LLG-0824-GBC** — Green Band Canonization: threshold drift as a form of distributed contradiction
- **FREF-0635-WWLV** — Witness Without Leverage: a custody gap may produce multi-layer explanation residue when each layer files a locally valid account of the same non-operative testimony

---

### Archive Position

Explanation layer residue is what accumulates when an institution is too
orderly to lie and too layered to tell the whole truth in any single place.

One smooth explanation is a compression failure.
A stack of smooth explanations is a routing failure.
The distinction is whether a careful reader could find the contradiction by
reading harder, or whether the stack itself has made it unfindable.

This audit preserves the stack. It does not flatten it.
