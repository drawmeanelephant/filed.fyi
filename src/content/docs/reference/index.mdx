## What Lives Here

The **reference** directory contains:

- structural doctrine (taxonomy definitions, directory maps, naming conventions),  
- subsystem nodes (BHDSS, SBI annex, Directive Conflict Stack, Synthetic Affect Suite, Forms Drift, etc.),  
- navigation aids that pretend this is an information architecture instead of a graveyard.[file:10]

This layer is where the archive explains how it thinks it works.  
Everything else is where it proves otherwise.

---

## How to Read It

Reference docs fall into two broad species:

- **Doctrine Sheets** — MAP, SAC, SSP, and similar attempts to codify behavior into categories.[file:10]  
- **Reference Nodes** — overviews like BHDSS or Synthetic Affect & Successor Suite that summarize clusters of incidents in plain language.[file:10]

They are useful for:

- orienting new readers,  
- reminding old readers what the words used to mean,  
- giving future investigators a starting map for the decay.

Cross-links point down into Lorelog, mascots, posts, haiku, and limericks.  
Nothing in reference is allowed to exist without at least one dependent ghost.

## What This Index Is For

This page is the **table of contents for how to pretend this is organized**:

- list of top-level nodes (BHDSS, silence annex, directive stack, synthetic affect, forms drift, metrics/optics, breeding program),  
- pointers to doctrine documents that those nodes summarize,  
- brief notes on what each node is trying to tame.

If you are about to add a new cluster of weirdness, you have two jobs:

1. File the incidents.  
2. Add a reference node here that admits what you just did.

Everything gets filed.  
Not everything gets found.  
This layer exists to narrow the gap without closing it.
